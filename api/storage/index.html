storage API

The storage API are javascript functions that allow read / write operations.
Developers can then choose where to save files, using whatever backend language and environment they prefer.
There are currently two types of API, from which programmers can take inspiration: in the first, the functions call http APIs written in Node.js.
In the second, the application writes to the browser's API caches.
The APIs used are the following:

  • Create
  • Read
  • Update
  • Delete
  • Copy
  • Search
  • GetSettings
  • SetSettings
  • CreateComponent
  • DeleteComponent
It is worth noting that there is no need to write all these functions: If we decide to remove the package manager panel, we could do without the createComponent, deleteComponent, setSettings and Create api. In this way, we could work on a single project and the components would be "fixed" (neither importable nor writable directly by the user). This solution may be cost effective in some scenarios.
